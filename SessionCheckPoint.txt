  <technical_details>
  - Architecture now follows: GUI -> `ThuocClientController` -> socket `Request` -> `CommandDispatcher` -> `ThuocCommandHandler` -> service/repository.
  - Request protocol uses string command keys (`AUTH.*`, `THUOC.*`) rather than single global enum dispatch.
  - `NetworkClient` is persistent per app session and synchronized for request/response; currently synchronous call style.
  - Dispatcher issue fixed: changed immutable `Map.of(...)` to mutable map initialization to avoid runtime failure when registering.
  - Lot-level retrieval implemented with JPA query joining `ChiTietLoThuoc` + `Thuoc` + `LoThuoc` (+ eager joins for unit/manufacturer), filtered active
  and `soLuong > 0`.
  - New lot DTO: `ThuocTheoLoView` with `maLo`, `hanSuDung`, `soLuongLo` etc.
  - `CapNhatThuocPanel` no longer uses `ThuocJpaDAO` directly; all CRUD actions go through network responses.
  - Important environment quirk: no `mvn` and no `javac` available in PATH, so verification remains manual/static.
  - There are many unrelated existing worktree changes (including `target/` artifacts and other modified files); no cleanup/revert performed.
  - Potential unresolved consistency: both old and newer CRUD-related method shapes were changed across layers during iterations; code should be
  recompiled in a proper dev environment to confirm no signature drift remains.
  </technical_details>
